========================================================================
                         KEY FEATURES
========================================================================

1. 4-STEP WIZARD:
   Step 1: Basic information (ID, name, description, author)
   Step 2: Capability configuration (add/remove/edit)
   Step 3: Permission selection (multi-select)
   Step 4: Review and download

2. TEMPLATE GENERATION:
   - Generates 7 files: manifest.json, handlers.py, README.md,
     install/plan.yaml, docs/USAGE.md, icon.svg, .gitignore
   - In-memory ZIP creation (no temp files)
   - Generation time: < 100ms
   - File size: 5-15 KB

3. VALIDATION:
   - Extension ID format: [a-z0-9]+\.[a-z0-9]+
   - Required fields enforcement
   - At least one capability required
   - Client-side and server-side validation

4. API ENDPOINTS:
   GET  /api/extensions/templates
   GET  /api/extensions/templates/permissions
   GET  /api/extensions/templates/capability-types
   POST /api/extensions/templates/generate
